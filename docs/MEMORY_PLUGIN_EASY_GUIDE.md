# Memory Plugin 쉬운 설명서

## 한 줄 요약

> AI 에이전트가 **이전 대화를 기억하고 떠올릴 수 있게** 해주는 플러그인입니다.

---

## 왜 필요한가?

AI는 기본적으로 **대화가 끝나면 내용을 잊어버립니다.**

Memory 플러그인이 하는 일:
- 중요한 대화 내용을 **파일에 저장** (기억하기)
- 나중에 필요할 때 **파일에서 찾아서 읽기** (떠올리기)

```
사용자: "내 생일은 3월 15일이야"
AI: (메모리에 저장)

--- 며칠 후 ---

사용자: "내 생일 언제였지?"
AI: (메모리에서 검색) → "3월 15일입니다!"
```

---

## 핵심 개념 3가지

### 1. 한 자리만 있는 의자 (Exclusive Slot)

메모리 플러그인은 **동시에 하나만** 사용할 수 있습니다.

```
memory-core (기본)  ←── 지금 앉아 있음
my-plugin           ←── 앉으려면 memory-core가 일어나야 함
```

설정으로 자리를 바꿀 수 있습니다:

```json
{
  "plugins": {
    "slots": {
      "memory": "my-plugin"    ← 이걸로 바꾸면 my-plugin이 앉음
    }
  }
}
```

### 2. 필요할 때만 깨어남 (Lazy Loading)

메모리 플러그인은 게이트웨이가 시작될 때 바로 로드되지 않습니다.
AI가 **처음으로 기억을 찾으려 할 때** 그제서야 로드됩니다.

```
게이트웨이 시작 → (메모리 플러그인은 아직 잠자는 중)
    ...
사용자가 질문 → AI가 memory_search 호출 → (이때 깨어남!)
    ...
이후로는 계속 깨어 있음
```

### 3. 자동 저장 (Memory Flush)

대화가 길어지면 AI의 **기억 공간(컨텍스트)이 가득** 찹니다.
가득 차기 전에 중요한 내용을 자동으로 파일에 저장합니다.

```
대화 시작 ─────────────────────────────────── 대화 끝
[                    컨텍스트 윈도우                    ]
[■■■■■■■■■■■■■■■■■■■■■■■■■■■■■░░░░░░]
                                    ↑
                              여기쯤에서 자동 저장!
                              (memory/2026-04-02.md 에 기록)
```

## 플러그인이 하는 일 4가지

### (1) 도구 등록 — AI에게 기억 능력 부여

AI가 사용할 수 있는 **도구(tool)**를 만들어 줍니다.

| 도구 | 하는 일 | 비유 |
|------|--------|------|
| `my_memory_search` | 저장된 기억에서 검색 | 노트에서 키워드 찾기 |
| `my_memory_store` | 새로운 기억 저장 | 노트에 메모하기 |

```typescript
// AI가 이런 식으로 도구를 호출합니다
memory_search({ query: "사용자 생일" })
// → { key: "birthday", content: "3월 15일" }
```

### (2) 프롬프트 안내 — AI에게 사용법 알려주기

AI의 시스템 프롬프트에 **"이런 도구가 있으니 이렇게 써라"** 라는 안내를 넣습니다.

```
## Custom Memory
Use my_memory_search to recall prior context before answering.
Use my_memory_store to save important information.
```

이 안내가 없으면 AI는 도구가 있어도 **언제 써야 할지 모릅니다.**

### (3) 메모리 런타임 — 실제 저장/검색 엔진

도구가 호출되면 **실제로 데이터를 저장하고 찾는** 엔진입니다.

```
도구 호출 → 런타임 → 저장소(파일, DB 등)
```

### (4) 플러시 계획 — 자동 저장 규칙

컨텍스트가 가득 차기 전에 **언제, 어디에, 어떻게** 저장할지 정합니다.

```
- 언제: 토큰이 임계치를 넘으면
- 어디에: memory/2026-04-02.md
- 어떻게: 기존 내용에 이어쓰기 (덮어쓰기 X)
```

## 파일 구조와 역할

```
my-plugin/
├── index.ts              ← 여기서 4가지를 등록
│                            "나는 이런 기능이 있어요"
│
├── src/
│   ├── tools.ts          ← 도구 구현
│   │                        "검색해줘" "저장해줘" 처리
│   │
│   ├── prompt-section.ts ← AI에게 보여줄 안내문
│   │                        "이 도구는 이렇게 쓰세요"
│   │
│   └── runtime.ts        ← 메모리 엔진
│                            실제 저장/검색 로직
│
├── openclaw.plugin.json  ← 플러그인 신분증
│                            "나는 memory 종류입니다"
│
└── package.json          ← 패키지 정보
```

## 동작 순서 (처음부터 끝까지)

```
① 게이트웨이 시작
   └─ "my-plugin 이라는 memory 플러그인이 있구나" (발견만 함)

② 설정 확인
   └─ "plugins.slots.memory = my-plugin 이니까 이걸 쓰자"
   └─ "memory-core 는 꺼야지"

③ 사용자가 대화 시작
   └─ (아직 메모리 플러그인은 안 깨어남)

④ AI가 과거 기억이 필요한 질문을 받음
   └─ memory_search 도구 호출
   └─ 이때 처음으로 메모리 플러그인 로드!
   └─ 검색 결과를 AI에게 전달

⑤ AI가 중요한 정보를 발견
   └─ memory_store 도구로 저장

⑥ 대화가 길어져서 컨텍스트가 가득 참
   └─ 자동 플러시 트리거
   └─ memory/2026-04-02.md 에 중요 내용 저장
   └─ 컨텍스트 압축 (오래된 대화 요약)

⑦ 다음 대화에서
   └─ memory_search 로 이전에 저장한 내용 검색 가능
```

## 내 플러그인 활성화하는 법

OpenClaw 설정 파일에 한 줄만 추가하면 됩니다:

```json
{
  "plugins": {
    "slots": {
      "memory": "my-plugin"
    }
  }
}
```

이렇게 하면:
- 기존 `memory-core` → 비활성화
- `my-plugin` → 활성화

## 지금 구현된 것 vs 앞으로 할 것

| 항목 | 현재 상태 | 설명 |
|------|----------|------|
| 플러그인 구조 | ✅ 완료 | 패키지, 매니페스트, 엔트리포인트 |
| 도구 등록 | ✅ 완료 | search, store 도구 등록 |
| 프롬프트 안내 | ✅ 완료 | AI에게 도구 사용법 안내 |
| 런타임 | ⚠️ 스텁 | 인메모리 Map (재시작하면 사라짐) |
| 검색 엔진 | ⚠️ 기본 | 단순 키워드 매칭 |
| 플러시 계획 | ❌ 미구현 | 컴팩션 연동 자동 저장 |
| 영구 저장 | ❌ 미구현 | 파일/DB에 영구 저장 |
| 임베딩 검색 | ❌ 미구현 | 벡터 기반 시맨틱 검색 |

다음 단계는 **런타임에 실제 저장소를 연결**하는 것입니다.
(SQLite, Redis, 파일시스템, Pinecone 등)
